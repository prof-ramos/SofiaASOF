# SOFIA — Agente de IA para ASOF

> SOFIA (Suporte Orientado às Funções e Interesses dos Associados) é um chatbot com RAG (Retrieval Augmented Generation) desenvolvido para a ASOF — Associação Nacional dos Oficiais de Chancelaria do Serviço Exterior Brasileiro. Orienta Oficiais de Chancelaria sobre carreira, direitos, deveres e procedimentos funcionais. Stack: Next.js 16 (App Router) + Vercel AI SDK v3 + OpenAI GPT-4o + Supabase pgvector. Deployado na Vercel (Free tier). Este projeto é desenvolvido, testado, publicado e melhorado 100% por agentes de IA (Claude Code, codex-cli).

**Fluxo RAG resumido:**

```
Pergunta → Middleware (rate limit) → /api/chat → Embedding (text-embedding-3-small)
→ Supabase RPC sofia_match_documents (threshold=0.7, top=8)
→ Re-ranking heurístico → Otimização de contexto (2000 tokens, 3–5 chunks)
→ GPT-4o streaming → Resposta
```

## Pipeline RAG

- [Core RAG](lib/rag.ts): `generateEmbedding()`, `retrieveContext()`, `buildContextPrompt()`, `retrieveContextBatch()`
- [Re-ranking](lib/rag-rerank.ts): `rerankSources(query, sources)` — bônus por keywords, penalidade por tamanho, score por posição; stub pronto para cross-encoder (Xenova)
- [Otimizador de contexto](lib/context-optimizer.ts): `buildDynamicContextPrompt(sources, options)` com `maxContextTokens` — respeita limite de 2000 tokens, min 3 / max 5 chunks, máx 2 chunks por documento
- [System prompt da SOFIA](lib/system-prompt.ts): personalidade, regras de citação legal, hierarquia de fontes, restrições de linguagem
- [Métricas](lib/metrics.ts): contagem de tokens via tiktoken (cl100k_base), preços por modelo, logging no Supabase

## Convenções para Agentes de IA

**Regras críticas — leia antes de modificar qualquer arquivo:**

1. **Nunca use "diplomacia"** no system prompt ou em qualquer texto visível ao usuário. Use sempre "Serviço Exterior Brasileiro".
2. **RPC correta:** `sofia_match_documents` (não `match_documents` nem `sofia_match_docs`).
3. **tiktoken é WASM** — jamais remova `serverExternalPackages: ['tiktoken']` do `next.config.ts`.
4. **Limite Vercel:** `maxDuration = 30` em `/api/chat` — não aumentar (Free tier).
5. **RAG threshold:** manter `>= 0.6`; o padrão é `0.7`. Abaixo disso aumenta ruído.
6. **Rate limiting duplo:** in-memory (`lib/rate-limit.ts`) + Supabase (`rate_limit_entries`). Não remover nenhuma camada.
7. **Logger:** use `logger` de `lib/logger.ts` em vez de `console.*` (silenciado em produção, exceto `logger.error`).
8. **Validação Zod:** sempre chamar `safeValidateChatRequest()` antes de processar qualquer payload.
9. **Degradação graciosa:** falhas em RAG/embedding retornam `[]` e continuam sem contexto — nunca lançam exceção para o usuário.
10. **Paralelização:** embedding e conversão de mensagens devem ocorrer simultaneamente via `Promise.all`.

**Padrões de código:**

```typescript
// Rate limiting (app/api/chat/route.ts)
import { rateLimit } from '@/lib/rate-limit'
const { isRateLimited, reset } = rateLimit(clientId, { interval: 60000, limit: 20 })

// Validação (app/api/chat/route.ts)
import { safeValidateChatRequest } from '@/lib/validation/schemas'
const result = safeValidateChatRequest(body)
if (!result.success) return new Response(..., { status: 400 })

// RAG com degradação graciosa (app/api/chat/route.ts)
const sources = await retrieveContext(query).catch(() => [])

// Re-ranking + otimização (app/api/chat/route.ts)
const reranked = await rerankSources(query, sources)
const contextPrompt = buildDynamicContextPrompt(reranked, { maxContextTokens: 2000, minChunks: 3, maxChunks: 5, diversityThreshold: 2 })

// Streaming (app/api/chat/route.ts)
const result = streamText({ model: openai('gpt-4o'), system: SYSTEM_PROMPT, messages })
return result.toUIMessageStreamResponse()
```
